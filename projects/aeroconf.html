AeroConf: Aircraft Configuration Language

Developing a robust DSL for aircraft configuration validation.

Project Overview

The AeroConf project involved developing a sophisticated Domain-Specific Language (DSL) and parser specifically for defining, validating, and generating reports for civil aircraft configurations. This tool addresses the complex challenge of ensuring aircraft designs adhere to intricate structural, operational, avionics, engine, and compliance data specifications.

My Role

As the primary developer, I was responsible for the entire lifecycle of the DSL and parser, from grammar design to GUI implementation.

Technologies Used

  • Python
  • ANTLR (for parser generation)
  • GUI Framework(streamlit and Tkinter)

Key Features

  • Comprehensive Grammar: Designed a grammar supporting structural, operational, avionics, engine, and compliance data.
  • GUI Tool: Built an intuitive GUI to load .acl files, validate configurations, and provide instant feedback.
  • Robust Parsing: Integrated error handling and token-based parsing for reliable configuration management.
  • Report Generation: Capability to generate reports based on validated configurations.

The Process

My approach involved several key phases:

  1. Grammar Definition: Collaborated on defining the syntax and semantics for the .acl (Aircraft Configuration Language) using ANTLR.
  2. Parser Development: Implemented the parser using Python to process and interpret .acl files.
  3. Validation Logic: Developed the validation rules to ensure configurations met specified criteria.
  4. GUI Development: Designed and built the graphical user interface for seamless user interaction and real-time feedback.
  5. Error Handling: Implemented robust error detection and reporting to guide users in correcting invalid configurations.
